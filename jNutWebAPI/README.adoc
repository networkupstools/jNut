jNutWebAPI Web Application
~~~~~~~~~~~~~~~~~~~~~~~~~~

This directory contains source files for the jNutWebAPI web application,
which is a Java web application (WAR) to access NUT server(s).

You can deploy it in any web archive container (like tomcat) to access
NUT's `upsd` data server in a simple way, without having to know the NUT
protocol or manipulate `nut-scanner` if installed locally.

jNutWebAPI building requirements
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

jNutWebAPI requires to be built:

- A Java JDK 6 correctly set in environment (i.e. `bin` folder in `PATH`)
- A Maven 3 installation set in environment (i.e. `bin` folder in `PATH`) with
  sufficient configuration (internet connection, local repository) to let maven
  get all plugins to make processes.
- jNut built and installed in the local maven repository (see jNut `README.adoc`).

jNutWebAPI is written in Java SE 6. Settings in `pom.xml` are however
higher (via properties you can change on ancient systems) so it is easy
to build by default on modern toolkits.

jNutWebAPI building
^^^^^^^^^^^^^^^^^^^

Once JDK and Maven installed and configured, go into the jNut directory
and type:

	mvn install

The produced package called `jNutWebAPI-xxx-SNAPSHOT.war` is located in
`target` subdirectory.


Workspace cleaning
^^^^^^^^^^^^^^^^^^

The jNutWebAPI build workspace can be cleaned by removing the `target`
subdirectory or by typing:

	mvn clean

Deloyment and Web access
^^^^^^^^^^^^^^^^^^^^^^^^

The following is dedicated to a local Apache Tomcat server listening on TCP
port '8080' with management access rights correctly set.

You can deploy the jNutWebAPI in tomcat by uploading the war file through the
"deploy" section of the manager page accessed at http://localhost:8080/manager
of your server. Then you can access the webapp at an URL like:

	http://localhost:8080/jNutWebAPI-0.2-SNAPSHOT/nut

You have two parts :

- NUT `upsd` data server communication with the "URI subdirectory" `/servers`
- `nut-scanner` interface with the "URI subdirectory" `/scan`

UPSD communication
^^^^^^^^^^^^^^^^^^

Accessed at http://localhost:8080/jNutWebAPI-0.2-SNAPSHOT/nut/servers and
subpaths, you can query some data from upsd.

In the examples below, '{host}', '{device}', '{variable}' represent host,
device and variable names.

Discovered pieces of information are reported in the form of a valid
JSON object or array. Examples follow:

* query the device list registered in the `upsd` data server of the
  specified host:

	.../nut/servers/{host}

* query the device description:

	.../nut/servers/{host}/{device}

* query available variables of the specified device:

	.../nut/servers/{host}/{device}/vars

* query the specified variable value:

	.../nut/servers/{host}/{device}/vars/{variable}

nut-scanner interface
^^^^^^^^^^^^^^^^^^^^^

Accessed at http://localhost:8080/jNutWebAPI-0.2-SNAPSHOT/nut/scan with some
optional parameters, it basically wraps access to a locally installed
`nut-scanner` executable (`/usr/local/ups/bin/nut-scanner` by default).

It will return a list with succinct descriptors of devices with `driver` and
`port` values (in the NUT configuration sense).

You can specify a start and end IP range for networked device discovery
by specifying parameters:

	.../nut/scan?start=192.168.1.1&end=192.168.1.24

or a CIDR mask:

	.../nut/scan?mask=192.168.1.1/24
